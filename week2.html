Неделя 2 - Детский сад

День 1. А что такое CSS?

CSS (стили или таблицы стилей)- используется для того что бы управлять дизайном сайта и оформлением каждого элемента страницы.

Способы заведения стилей:

Связанные стили
Они находятся внутри отдельного файла стилей, и связываются со всеми страницами сайта при помощи специального тега <link>
Глобальные стили
Они находятся на одной странице с HTML и задаются в теге <head>
Внутренние стили
Задаются внутри тега при помощи специального атрибута style

Этот текст мы сделали красным про помощи внутренних стилей, задав следующий атрибут:style="color: red"

Глобальные стили задаются при помощи тега <style></style> Он сообщает браузеру, что внутри него идет CSS.

Если у вас сайт состоит из множества страниц - лучше всего подойдут связанные стили. Глобальные стили используются на одностаничниках.

Важно:

У каждого способа задания стилей есть свой приоритет. Самый высокий приоритет у внутренних стилей. Затем идут глобальные стили. И самый низкий приоритет у связанных стилей.

В CSS, как и в HTML есть свои комментарии. Они задаются про помощи: /* текст */

Синтаксис CSS

Как HTML состоит из тегов, так CSS состоит из селекторов. Селекторы указывают к какому элементу или элементам сайта мы хотим применить стили.

В свою очередь каждый селектор состоит из свойств, а каждое свойство состоит из значений. Вот базовый синтаксис:

Пример использования

body {color:#272727;}

селектор {свойство: значение;}

У каждого селектора может быть несколько свойств - ограничений по количеству нет. Свойства задаются через точки с запятой.

Селектор {свойство: значение; свойство: значение2; свойство: значение3;}

Пример использования

body {color: #272727; background; #efefef;}

И у каждого свойства может быть несколько значений. Значения указываются через пробел.

селектор{свойство: значение значение2 значение3 значение4;}

Пример использования

body {color: #272727; background: #efefef url(images/bg.png);}

День 2. Селекторы

Они используются для того, чтобы конкретно указать элемент сайта, к которому мы собираемся применить стили.

Для того, что бы наши гадкие ручонки могли добраться до любого!!! Элемента сайта мы будем использовать различные виды селекторов.

Виды селекторов.

Селекторы тегов - используются для того, что бы конкретно указать теги, к которым мы хотим применить стили. В качестве названий используются имена тегов (без угловых кавычек).

Пример использования

h2 {border-bottom: 1px dashed #ccc;}
для всех заголовков 2 уровня задали штрихпунктирную границу снизу толщиной в 1 пиксель.

Классы (селекторы классов) - нужны для того, что бы применять стили к отдельным элементам сайта выборочно, а не ко всем сразу.

В файле стилей задаются следующим образом:

.имя_класса {стили для него}

В HTML привязываются к тегам при помощи специального атрибута class="имя_класса"

Синий текст Красный текст Черный текст Зеленый текст

Идентификаторы (селекторы идентификаторы) - нужно для того, что бы применять стили к отдельным элементам сайта выборочно, а не ко всем сразу. Отличаются от классов тем, что они должны быть уникальны и могут использоватся только ОДИН раз на странице>

В файле стилей задаются следующим образом:

#идентификатор {стили для него}

В HTML привязываются к тегам при помощи специального атрибута id="идентификатор"

Синий текст Красный текст Черный текст Зеленый текст

Важно:

Имена для идентификаторов и классов могут быть произвольные, однако есть свои ограничения. В именах могут использоватся только латинские буквы, а также знак дефиса и нижнего подчеркивания. Так же могут использоватся цифры. И главное - имя должно начинаться с латинской буквы.

Важно:

Помните, что у идентификаторов приоритет всегда выше, чем у классов.

Универсальный селектор- применяет стили сразу для всех элементов сайта.

Выглядит он следующим образом:

* {стили}

Селекторы атрибутов - позволяют задать стили тегам у которых есть определенные атрибуты.

Задаются через квадратные кавычки, в которых указывается имя атрибута.

Вот как это выглядит:

[атрибут]{стили}

Контекстные селекторы

Указывают, что необходимо задавать стили для тегов, которые находятся внутри других тегов. Задаются через пробел.

Фишка: вы можете использовать несколько вложений селекторов.

Дочерние селекторы

Это селекторы, которые находятся внутри другого селекторов, причем внешний селектор называется родителем. Задаются через угловую кавычку {знак больше}

День 4. Стройбаны

Два великих тега!

<div> - это пустой тег бочного вида. Который надо использовать для структуры сайта.

<span> - это пустой тег строчного вида. Который мы можем использовать для оформления отдельных участков текста.

Блочные элементы - это теги, которые по умолчанию растягиваются на всю ширину родителя. Начианются с новой строки, и следующий за ними текст тоже переносят на новую строку.

Строчные элементы - это теги, которые не переносят на новую строку.

Aliquam posuere quam a felis tincidunt, et pretium lacus cursus. Aliquam justo tortor, ornare et viverra id, consectetur in neque.

Важно: блочные элементы автоматически растягиваются на всю ширину родителя. Высота у них по умолчанию зависит от содержимого.

Обтекание

Оно задается при помощи float. Если есть обтекание, то тег, которому мы его задали - перестает переносить элементы после него на новую строку. И эти элементы начинают его обтекать.

Важноесли блочному элементу задать обтекание - он перестанет растягиваться на всю ширину родителя.

Значения свойства float:

none - оно присваевается по умолчанию.

left - она прижимает элемент к левому краю, и все остальные теги, которые идут за ним - обтекают его с провой стороны

right - оно работает наоборот

Очистка обтекания

За очистку обтекания отвечает свойство clear. Когда мы его задаем, все теги начиная с тега с этим свойством перестают обтекать.

Значение свойства clear

none - очистки нет

left - очистка обтекания тегов со свойством float:left;

right - очистка обтекания тегов со свойством float:right;

both - очистка всех обтеканий

День 5. Измерители

В стилях есть свойства, которые влияэют на размер блоков.

Длинна и высота

Свойство height - указывает на высоту блока.

Свойство width - указывает ширину блока.

Важно:помните, что размеры мы можем указывать только для блочных типов элементов. Для строчных они работать не будут.

Значения и еденицы измерения:

auto - задается значение высоты по умолчанию. Ширина блочных элементов зависит от родителя

% - указывают размер блока в процентах. Размер вычисляется относительно родителя.

px/in/pt - количественные, точные значения.

Важно:если в картинке прописать только одну какуюто сторону, а для второй указать auto, то она автоматически выровняется согласно пропорциям.

Границы

В первую очередь помните, что любая граница созданная при помощи свойства border влияет дополнительно на размер в зависимости от своей толщины.

border - указывает границу, по умолчанию она отсутствует у многих элементов сайта.

Синтаксис

border: толщина тип цвет;

Толщину можно указывать любую в количественных еденицах измерения.

Типы границ:

dotted dashed solid outset inset groove double ridge

Ширина этого блока указана в 300 пикселей. Однако она на самом деле составляет 320 пикселей, так как у этого блока задана граница толщиной в 10 пикселей.

Как указывать границы отдельно:

border-top - граница сверху

border-right - граница справа

border-bottom - граница снизу

border-left - граница слева

У этого блока разные границы, потому что мы использовали специальные стили.

Поля и отступы

Поля указывают внутренние отступы, а отступы - внешние.

padding - это свойство задает внутренние отсупы или поля. Влияет на размер всех элементов, которым оно задано.

Поля можно указывать отдельно:

padding-top - Поля сверху

padding-right - Поля справа

padding-bottom - Поля снизу

padding-left - Поля слева

Можно указывать поля разными способами:

padding: вверх право низ лево

padding: вверх прево лево низ

padding: вверх низ право лево

Ширина этого блока 300 пикселей, но на самом деле у него ширина 322 пикселя. 40 пикселей за счет паддингов, и еще два за счет границ.

margin - задет внешние отступы. Они работают точно так же как и поля, за тем исключением, что они не влияют на размер блока.

Важно: для строчный элементов внешние отступы не работают по вертикали, только с права и с лева
