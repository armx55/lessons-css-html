День 2. Музыкальный проигрыватель

Музыкальный плеер на сайте

тег<audio> позволяет проигрывать музыкальные композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src.

<audio src="путь к картинке"> </audio> <audio> <sourcesrc="путь к картинке"></audio>

Атрибуты музыкального проигрывателя:

control - указывает, что необходимо отоброжать элементы контроля над плеером.

autoplay - задает автоматическое воспроизведение файла.

loop - зацикливает воспроизведение файла.

preload - указывает браузеру, как надо подгружать мультимедиа на странице.

  • none не загужать файл
  • autoстоит по умолчанию, файлы загружаются вместе со страницей
  • metadata - браузер подгружает только информацию о файле

Заключение:

С аудио и видео все понятно надо просто иногда повторять, чтобы не забыть.

День 3. Видеопроигрыватель

Видеопроигрыватель на сайте

тег<video> позволяет проигрывать видео композиции на сайте. Путь к странице можно задать двумя способами. Через тег <source>, либо через атрибут src.

<video src="путь к картинке"> </video> <video> <sourcesrc="путь к картинке"></video>

Атрибуты, которые мы рассмотрели к аудиопроигрывателю, подходят также и к видеопроигрывателю. Но у него есть еще несколько своих дополнительных атрибутов.

Атрибуты видеопроигрывателя

width - указывает ширину для видео в пикселях или в процентах.

height - указывает высоту для видео в пикселях или в процентах.

poster - вставляет постер (картинку-заставку) в видео, которая отображается до воспроизведения.

Помните, что пропорции постера и видео должны соответствовать, иначе постер будет отображаться непропорционально и не красиво. Будет искажаться.

<source> - указывает путь к файлу который необходимо проиграть. Ниже рассмотрим атрибуты этого тега

  • type - указывает тип воспроизводимого файла.
  • src - задает путь к файлу
  • media - для какого устройства должен производится этот файл.

Значение атрибута media

  • all - для всех устройст
  • handheld - для смартфонов, телефонов и иже с ними.
  • screen - для экранов монитора.
  • speech - для речевых ситезаторов.
  • projection - для проекторов.
  • tv - для теликов.
  • tty - для телетайпы.
  • print - для печати.

День 4. Формы

формы задаются при помощи тега <form>, который является парным. Содержимое формы на 99% состоит из тегов <input> различного типа. Этот тег является одиночным.

<form action="#" accept-charset="utf-8" method="post" name="test">
   <   <<input name="name">
   <   <<input name="mail">
   <   <<input name="text">
</form>

Атрибуты форм:

  • action - указывает на адрес программы или файла обработчика.
  • accept-charset - устанавливает кодировку для отправленных данных.
  • method - имеет два значения (get или post). Указывает как передавать данные.
  • name - задает уникальное имя для формы.
  • enctype - указывает способ кодировки отправленных данных
  • target - указывает каким образом будет отображон результат выполнения формы.
  • autocomplete - включает/выключает (on/off) автозаполнение форм.
  • novalidate - отменяет проверку на правильность заполнения полей форм.
Помните, что имя формы и имена ее элементов должны быть уникальными. Имена задаются произвольно, они могут состоять только из латинских букв, цифр и дефисов.

Почти все элементы формы задаются через тег <input> и без этого тега форма работать не сможет.

У тега input обязательно должны присутствовать 2 атрибута name и type.

День 5. Тег input

В HTML5 появилась множество новых полей. Будьте внимательны - некоторые новые поля еще некорректно работают в браузерах

Новые значения атрибута type в HTML5

  • color - поле для ввода цвета
  • email - указывает, что в поле должен быть введен адрес электронной почты
  • number - числовое поле
  • date - указывает, что в этом поле содержится дата
  • month - указывает месец
  • week - указывает неделю
  • datetime - указывает дату и время
  • datetime-local - указывает местное время и дату
  • time - указывает время
  • url - для ввода url
  • tel - поле для телефона
  • search - для поиска информации
  • range - задает ползунок

Будьте внимательны, так как не все типы полей поддерживаются современными браузерами. Обычно, если поле не поддерживается, оно обрабатывается браузерок как обычное текстовое поле.

Поля email, search, range, number, tel и url уже поддерживаются всеми современными браузерами. И вы можете использовать эти типы полей на своих сайтах. Не беспокоясь за работоспособность
